# OOP-Concepts-Object-Oriented-Programming-I
---

### Course Introduction
- **Primary focus is OOP theory** — every week leads with the *concept*, not the syntax
- **Java is a supporting tool** — code examples illustrate the concept, not teach the language
- **Deeper conceptual topics added** — Object Relationships (Week 12), SOLID Principles (Week 14), and Design Patterns (Week 15) which are core to any OOP course

---

### 📐 4-Part Structure

| Part | Weeks | Focus |
|---|---|---|
| Foundations | 1–3 | What OOP is, Classes, Objects, State |
| Four Pillars | 4–7 | Encapsulation, Constructors, Inheritance, Overriding |
| ⭐ Mid-Term | 8 | Weeks 1–7 |
| Advanced Pillars | 9–11 | Polymorphism, Abstract Classes, Interfaces |
| OOP in Practice | 12–15 | Relationships, Exceptions, SOLID, Patterns |
| ⭐ Final Exam | 16 | All topics |

---

### 📋 Every Week Includes
- **Core Concept** — plain English definition
- **Real-world analogy** — before any code
- **Conceptual diagram or comparison table** — ASCII
- **Java illustration** — to demonstrate the concept
- **Test Checkpoints**, **Common Mistakes**, **3-tier Extension Challenges**
